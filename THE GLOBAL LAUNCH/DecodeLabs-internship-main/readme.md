# 🚀 The Global Launch – Portfolio Website Deployment (AWS)

## 📌 Project Overview

The Global Launch is my first internship project at DecodeLabs, where I successfully deployed a professional portfolio website using Amazon Web Services (AWS) with industry-standard best practices.

This project demonstrates secure cloud deployment, storage management, version control, and implementation of security principles in a real-world environment.

## 🛠️ Technologies Used

- ☁️ Amazon S3 – Static Website Hosting  
- 🔐 IAM (Identity and Access Management) – Secure Access Control  
- 🌍 AWS Management Console  
- 📦 Git & GitHub – Version Control  
- 🧾 JSON Policies – Custom IAM Policy Configuration  

---

## 🏗️ Architecture Overview

- Created an S3 bucket configured for static website hosting  
- Enabled bucket versioning to maintain file history and rollback capability  
- Configured proper bucket policies for controlled public access  
- Implemented IAM users with Principle of Least Privilege (PoLP)  
- Managed project files using Git and GitHub for structured version control  
- Applied secure access practices to prevent unauthorized modifications  

---

## 🔐 Security Implementation

### ✅ Principle of Least Privilege (PoLP)

- Created IAM user with restricted access  
- Granted only required S3 permissions  
- Avoided root account usage  

### ✅ Bucket Versioning

- Enabled versioning for file recovery and rollback  
- Protected against accidental deletions  

### ✅ Secure Policy Configuration

- Used JSON-based bucket policies  
- Restricted write permissions  
- Followed AWS recommended security configurations  

### ✅ Access Management

- Disabled unnecessary public access  
- Used IAM roles instead of root credentials
